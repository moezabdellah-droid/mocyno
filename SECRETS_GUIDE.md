# GitHub Secrets Configuration Guide

To enable the CI/CD pipeline to build and deploy your application correctly, you must configure the following **Repository Secrets** in your GitHub repository settings (`Settings` > `Secrets and variables` > `Actions` > `New repository secret`).

## 1. Firebase Configuration (Mobile & Admin)
These values are required for the application to connect to Firebase at runtime (via `import.meta.env`). Grab these values from your local `.env` or Firebase Console.

| Secret Name | Description | Example Value (DO NOT COPY DIRECTLY) |
| :--- | :--- | :--- |
| `VITE_FIREBASE_API_KEY` | Firebase API Key | `AIzaSyDOCAbC123dEf456GhI789jKl012-MnO` |
| `VITE_FIREBASE_AUTH_DOMAIN` | Auth Domain | `mocyno-app.firebaseapp.com` |
| `VITE_FIREBASE_PROJECT_ID` | Project ID | `mocyno-app` |
| `VITE_FIREBASE_STORAGE_BUCKET` | Storage Bucket | `mocyno-app.firebasestorage.app` |
| `VITE_FIREBASE_MESSAGING_SENDER_ID` | Sender ID | `123456789012` |
| `VITE_FIREBASE_APP_ID` | App ID | `1:123456789012:web:abcdef123456` |

## 2. Firebase Deployment Token
Required for the GitHub Action to deploy to Firebase Hosting.

1. Run `firebase login:ci` on your local machine.
2. Copy the token printed in the terminal.
3. Add it as a secret:

| Secret Name | Description |
| :--- | :--- |
| `FIREBASE_TOKEN` | The CI token generated by `firebase login:ci` |

## Check CI/CD Logs
After adding these secrets, re-run the failed GitHub Action:
1. Go to the **Actions** tab in GitHub.
2. Click on the latest workflow run.
3. Click **"Re-run jobs"**.
